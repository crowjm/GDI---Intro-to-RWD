We're going to talk about mobile first, first.

What is Mobile First?

Mobile First started being talked about around 2010.

Rather than starting your design with the desktop, the idea of mobile first is to start designing and building for small screens, and grow from there.

Maybe draw this proces on a white board?
Mobile first fits really well something called progressive enhancement. (ask if anyone knows what that is). PE is the idea of starting with a comprehensive foundation for your site, and adding flourishes from there. This way your content can always be accessed, even if just HTML is rendered.

Mobile Means Different Things

There Are a Lot of Things You Can't Control

  • What device the user has.
  • What if the user's JavaScript doesn't load?
  • What if the user's CSS doesn't load?
  • The user's connection speed.

Designing for the LCD

More than just your content and design, Mobile First is about desiging and developing for the lowest common denominator.

With Mobile First, you also want to consider designing for devices that don't load your CSS and/or JS.

You want your site to be as performant as possible.

Mobile First means designing in a way that will make your content as accessible as possible for your users. All of 'em.

Benefits of Mobile First

  • Mobile First helps you simplify and prioritize your content.
  • Mobile First makes your site more perfomant and accessible.
  • Mobile First helps ensure a great user experience.
So now let's transition to RWD.

What is Responsive Web Design (RWD)?

RWD is part philosophy and part front end strategy. RWD is the crafting of websites that will be functional and beautiful regardless of which screen they're viewed on.

It originated in 2010, around the same time as Mobile First.

Combine RWD with Mobile First for a wonderfully built site.

The long and short of it is that we're designing for more devices, more input types, more resolutions than ever before. The web has moved beyond the desktop and it's not moving back.

‚ÄîEthan Marcotte

This quote is from Ethan Marcotte, who coined the term RWD back in 2010.
So this is really similar to Mobile First, in that the world has moved beyond the desktop. But RWD isn't just about ensuring a good mobile experience. It's about making sure your user can access your site and content no matter what device, screen size, resolution, etc. they're using. So phones, tablets, desktops, laptops, tvs-even watches and glasses.

Why Responsive Web Design?

...No design, fixed or fluid, scales seamlessly beyond the context for which it was originally intended.

‚ÄîEthan Marcotte

RWD allows us to create seamless experiences for our users that will work regardless of the device they're using.

Fluid Grid

Fluid Grid

  • In order to be responsive, we need to express our grid (containing and container elements) in flexible, relative, proportional terms (percentages).
  • TL;DR: % not px
  • This percentage will represent the width of the containing element, which means it will reflow according to viewport widths.

Fluid Grid

Formula: Target / Context = Result

Wherein...

  • Target is the div you want to find the % of
  • Context is parent div
  • Result is width as %

Flexible Media

Use percentages for images' (or gifs' or videos', etc.) dimensions so that they can grow and shrink as user adjusts browser.

Flexible Media

Media Queries

Media Queries

Media queries are a test for the browser. If conditions (like height, width, resolution) are met, certain styles will be shown. If not, browser will skip over.

Use media queries when you need them in the design, not at prescribed device viewport widths.

The point where a media query happens is a breakpoint.

There are a lot of types of media queries.

Media Queries

@media all

Applies to all media types: screen, print, tv...

Media Queries

@media print

For printers

Media Queries

@media screen and (resolution: #dpi)

For high resolution devices, like retina displays. You can use this to display higher res images to higher res devices.

So many!

@media all and (orientation: landscape)
@media all and (orientation: portrait)

There's even...

@media 3d-glasses

If you Google...

You may see...

@media only screen ...

"Only" is used so older browsers ignore the code.

This will only work in devices:

@media screen and (min-device-width)

Media Queries

@media screen

The one you want to use! Browsers and mobile devices are in this category.

We'll combine this with a min-width to change things about our design at certain sizes.

@media screen and (min-width: width)

Develop it!

Add a media query to your site that makes the background teal at 600px.

@media screen and (min-width: 600px) { body {background: teal;} }

Fluid Typography

As Mobile First and RWD gained traction, fluid typography came on the scene.

Fluid typography means your type adjusts as the screen size changes.

An awesome first step is using em instead of px. Ems are a unit of measurement that's relative to the base font-size.
